2.8. Automatic Reclustering

Besides the ability to manually reshape existing clusters, Klusters provides a mechanism to invoke an external automatic cluster cutting program such as Kenneth D. Harris' KlustaKwik to selectively recluster currently selected clusters.

Select Actions->Recluster. This will open a new tab displaying all the messages output by the reclustering program. Error messages are displayed in red, and normal output is displayed in blue. Depending on the reclustering program, output messages may also be saved to log files on the disk.

When reclustering completes, a dialog box pops up to inform you of the IDs of the newly created clusters. If you are unsatisfied with the results, as with any other operation reclustering can be cancelled by selecting Edit->Undo.

If for any reason you want to abort reclustering, simply select Actions->Abort Reclustering.

The reclustering program and its parameters can be configured in the Settings Dialog.

Reclustering is a processor-intensive task that may take a substantial amount of time to complete if too many clusters are selected at once. Typically, reclustering should be performed on one to three clusters.
